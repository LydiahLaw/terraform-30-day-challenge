# 30-Day Terraform Challenge

A structured, hands-on challenge to build real infrastructure-as-code skills using Terraform and AWS over 30 days. Every day has a dedicated folder with code, notes, and documentation.

Repository: [LydiahLaw/terraform-30-day-challenge](https://github.com/LydiahLaw/terraform-30-day-challenge)

---

## Why I am doing this

I am a career transitioner moving from Finance into Cloud and DevOps Engineering. I have been building hands-on skills through the StegHub DevOps apprenticeship, the Tech4Dev Cloud and DevOps Fellowship, and AWS re/Start — and Terraform has come up consistently as a core tool in every serious infrastructure role I am targeting.

This challenge is my way of going deep. Not just running commands, but understanding what Terraform is doing, why it is designed the way it is, and how to use it to build infrastructure that is reliable, repeatable, and production-ready. I am also documenting everything publicly because I learn better when I write, and because I want the work to speak for itself.

---

## Tools and Tech Stack

| Tool | Purpose |
|---|---|
| Terraform v1.14.7 | Infrastructure provisioning and IaC |
| AWS (Free Tier) | Cloud provider |
| AWS CLI | Authenticating and interacting with AWS from the terminal |
| IAM | Scoped credentials for Terraform |
| Visual Studio Code | Editor, with HashiCorp Terraform and AWS Toolkit extensions |
| Git and GitHub | Version control and public documentation |

## Structure

Each day has its own folder containing Terraform configuration files, a day-specific README with tasks completed, commands used, and lessons learned.
```
terraform-30-day-challenge/
├── day-01/
├── day-02/
├── ...
└── day-30/
```
